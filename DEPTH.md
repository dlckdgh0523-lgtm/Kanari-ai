# 심화 기획 (MVP -> 운영 가능 수준)

> 목표: 이력에 "썼다"고 쓰기 위한 장식이 아니라, 실제로 돌릴 수 있는 깊이.
> 각 항목은 "왜"와 "면접에서 뭐라 답하나"를 함께 적는다.
> 작성일: 2026-07-12

## 실행 순서 (의존성 기준)

테스트로 현재 동작을 잠근 뒤에 큰 리팩터링을 한다. 순서를 지킨다.

```
D1 테스트 기반 + CI     ← 지금부터. 이후 리팩터링의 안전망
D2 Redis 도입           ← 분산 상태(쿨다운/dedup/rate-limit)의 토대 + 수평 확장
D3 Kafka 심화           ← DLQ, 재시도, 멱등성(Redis), 컨슈머 랙
D4 카디널리티 방어       ← 폭주 클라이언트로부터 DB 보호
D5 Winston 심화         ← child logger(ALS), profiling, exceptionHandlers
D6 보안 강화 (전부)      ← JWT fail-fast, refresh 회전, PII 스크러버, 부하테스트 도메인검증, 키별 쿼터, 감사로그
D7 엔지니어링 부채       ← 마이그레이션, 페이지네이션, health/readiness, graceful shutdown
D8 AI 조사 에이전트      ← 간판. 증거수집 스캐폴딩은 키 없이, LLM은 키 오면
```

---

## D1. 테스트 기반 + CI

- **왜**: 신뢰성 감시 도구가 자기 테스트 0개인 건 자기모순. 큰 리팩터링(마이그레이션/Redis/DLQ) 전에 회귀 방지망이 필요.
- **무엇을**:
  - 단위: `fingerprint`(같은 버그 다른 줄, 메시지 정규화, 스택 없음), `p95FromBuckets`, `source-location` 파서
  - 통합: ingest→group 흐름(실제 DB), 인증 401/403, 소유권, 회귀 흐름, 동시성 경합
  - SDK: transport 배치·fire-and-forget(전송 실패해도 앱 안 죽음)
  - CI: GitHub Actions에서 lint+build+test. 우리가 만든 배포 게이트 철학을 우리 리포에 도그푸딩
- **면접**: "테스트 있나요?" → 순수 함수 단위 + mock 없는 실제 DB 통합(Welive에서 한 방식) + CI 게이트

## D2. Redis 도입

- **왜**: 지금 쿨다운/롤백 알림 상태가 워커 메모리(Map). 워커 재시작하거나 2대로 늘면 상태가 깨져 알람 중복. 분산 상태 저장소가 필요. 진로나침반·Welive의 Redis 경험 재사용.
- **무엇을**: 급증/롤백/지연 쿨다운, 멱등성 dedupeKey(재소비 방어), 인입 rate limit을 전부 Redis로. 이걸로 워커 수평 확장이 열린다.
- **면접**: "워커 2대로 늘리면?" → Redis에 공유 상태를 둬서 어느 워커가 처리해도 쿨다운·중복이 일관

## D3. Kafka 심화

- **왜**: 지금 깨진 메시지는 console.error 찍고 유실. 브로커 1대. 컨슈머 랙 안 봄. 진짜 파이프라인이라 하기엔 얕다.
- **무엇을**:
  - DLQ(`kanari.events.dlq`): N회 재시도 후 실패 메시지를 이유·횟수와 함께 격리. 유실 대신 보존
  - 지수 백오프 재시도 (포이즌 메시지가 뒤를 막지 않게)
  - 멱등 처리: 메시지 id를 Redis로 dedupe → 재소비(at-least-once)에도 중복 저장 안 함
  - 컨슈머 랙을 Prometheus로 노출 → 파이프라인이 밀리는 걸 Grafana에서 봄
  - 파티션 전략 문서화(key=projectId로 프로젝트 내 순서 보장), 2 워커 리밸런싱 시연
- **면접**: DLQ / 컨슈머 랙 / 멱등성 / 리밸런싱 — 이 네 단어를 자기 코드로 설명

## D4. 카디널리티 방어

- **왜**: 실제 공격/버그 시나리오. 버그난 SDK 사용자가 에러 메시지에 타임스탬프를 넣으면 그룹이 수백만 개로 폭발해 우리 DB가 죽는다. 라우트도 마찬가지.
- **무엇을**: 프로젝트당 분당 신규 그룹/라우트 수 상한. 초과하면 "high cardinality" 감지 + "other"로 접기 + 사용자에게 경고. 우리 자신의 카디널리티(프로젝트별 그룹 수)를 Grafana로 감시.
- **면접**: "폭주 클라이언트가 DB를 죽이면?" → 카디널리티 상한 + 접기 + 감지 알람

## D5. Winston 심화

- **왜**: "Winston을 다 쓰냐"에 정직하게 "transport만"이라 답했었다. 깊이를 채운다.
- **무엇을**:
  - SDK: `createLogger()` 헬퍼가 AsyncLocalStorage로 요청 컨텍스트(traceId)를 child logger에 자동 주입 → "붙이면 알아서"에 근접. profiling 헬퍼(구간 시간)
  - 카나리 자신: child logger로 요청별 컨텍스트, 커스텀 레벨, Winston exceptionHandlers/rejectionHandlers
- **면접**: "Winston 어디까지?" → transport + child logger(ALS 컨텍스트 전파) + profiling + exception handlers

## D6. 보안 강화 (전부)

- **왜**: 배포하면 공개 인터넷. 지금 구멍이 여럿.
- **무엇을**:
  - JWT_SECRET 미설정 시 운영에서 부팅 실패(fail-fast). 개발 폴백 금지
  - refresh 토큰 회전·해시 저장(진로나침반 패턴 재사용). 로그아웃/비번변경 전체 무효화
  - 부하 테스트 도메인 소유 검증(well-known 경로 프로브 or DNS) + 프로젝트당 쿨다운 → 공격 도구화 차단
  - 인입 키별 쿼터(지금은 IP 전역 throttler)
  - env를 부팅 시 zod로 검증, 없으면 즉시 실패
  - 콘솔 관리 액션 감사 로그(AuditLog)
  - SDK에 PII 스크러버: 이메일·토큰 패턴을 전송 전 마스킹(방어 심화)
- **면접**: 진로나침반에서 한 보안 설계를 SaaS 규모로 확장

## D7. 엔지니어링 부채

- **왜**: 운영 가능 수준의 기본기.
- **무엇을**:
  - TypeORM `synchronize:false` + 마이그레이션 체계(초기 생성 + 워크플로)
  - 커서 기반 페이지네이션(groups/events/logs)
  - `/health`(readiness) vs `/metrics` 분리, graceful shutdown(진행 중 메시지 처리 후 종료)
  - 부팅 시 설정 검증, DTO 검증 일관성
- **면접**: "운영 준비됐나?" → 마이그레이션/헬스체크/graceful shutdown/페이지네이션

## D8. AI 조사 에이전트 (간판)

- **왜**: 이 프로젝트의 창의적 전제. 지금 비어 있음.
- **무엇을**:
  - 트리거: 신규/회귀/급증에만(비용 게이트)
  - 증거 수집(키 없이 지금 가능): traceId 주변 로그 + 최근 배포 + 동시간대 다른 에러 + suspect commit(있음) + 유사 장애(있음)
  - 축(질문)별 분해 → 오케스트레이터 종합 → 회의론자 에이전트가 반증 → Discord 알람에 조사 리포트 첨부
  - 모델 다운시프트 + 프롬프트 캐싱(쇼핑 컨시어지 재사용), 단일 vs 멀티 A/B 측정
- **면접**: 멀티에이전트를 축별로 분해하고 반증으로 검증한 이유, 비용/품질 트레이드오프를 측정으로 답
- **의존**: Anthropic 키(창호 준비 시). 증거수집 스캐폴딩은 지금 만들고 LLM만 나중에 연결

## 원칙

- 각 D가 끝날 때마다: 테스트 통과 + 이 문서 체크 + 회고 블로그 한 편(면접 재료)
- 창호가 핵심 로직의 "왜"를 본인 언어로 설명할 수 있을 때 그 D가 진짜 끝난 것
- 넓히기 전에 깊이. 기능 추가보다 있는 걸 운영 가능하게
